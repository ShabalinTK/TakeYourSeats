# Take Your Seats (TUS) — Web Application

## Welcome
> *"Everything in software architecture is a trade-off. First Law of Software Architecture"*

**TUS** — это надежная и прозрачная веб-система контроля посещаемости для учебных заведений. Она решает проблему ложных отметок с помощью динамических QR-кодов и визуализирует рассадку студентов на интерактивных картах в реальном времени. TUS экономит лекционное время преподавателей и предоставляет администрации достоверную аналитику посещаемости.

---

## Business Case

Текущие методы контроля посещаемости в ВУЗах страдают от недостоверности (пересылка статичных кодов) и временных затрат на перекличку.

**Бизнес-цели:**
* **Достоверность:** Исключение возможности регистрации студентов вне аудитории.
* **Прозрачность:** Визуальная сверка карты приложения с реальной рассадкой студентов.
* **Эффективность:** Автоматизация формирования отчетности для преподавателей и администрации.

## System Requirements

### Stakeholders

* **SH-1: Преподаватель**: управляет учетными записями, создает сессии занятий и контролирует отчетность.
* **SH-2: Студент**: осуществляет регистрацию на занятии и бронирование физического места.

### Functional Requirements

* **UC-1: Процесс регистрации**:
    * Преподаватель запускает сессию и транслирует динамический QR-код.
    * Студент сканирует код, выбирает место на карте и подтверждает выбор кнопкой.
* **UC-2: Мониторинг**: преподаватель в реальном времени видит заполнение аудитории на интерактивной схеме.
* **UC-3: Отчетность**: автоматическая генерация отчетов по посещаемости в форматах Excel по завершении пары.

### Architecture Characteristics Requirements

* **QA-1: Security (Безопасность)**:
    * **Dynamic QR**: использование временных токенов в ссылках, которые делают пересылку кода бесполезной.
    * **Session Isolation**: каждая сессия уникальна для конкретной аудитории и временного интервала.
* **QA-2: Performance (Производительность)**: время отклика системы при выборе места должно быть минимальным (< 2с), чтобы не создавать «пробку» на входе.
* **QA-3: Usability (Удобство)**: адаптивный интерфейс с механизмом «выбор → подтверждение» для защиты от случайных нажатий.

---

## Constraints
* **CON-1**: Система должна быть доступна через веб-браузер без установки дополнительного ПО.
* **CON-2**: Необходимость предварительной интеграции планировок аудиторий в базу данных.

## Assumptions
* **ASM-1**: Веб-интерфейс является основной точкой взаимодействия и может обновляться независимо от серверной части.
* **ASM-2**: Студенты обеспечены персональными смартфонами с доступом в интернет.
* **ASM-3**: Аудитории оборудованы техническими средствами для демонстрации QR-кода (проектор/экран).
